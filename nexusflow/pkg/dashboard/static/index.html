    Guide
    
          
    NexusFlow is an autopilot layer for big Linux sockets: it sees NUMA “neighborhoods” and keeps work, memory, and caches aligned so jobs spend less time waiting on remote DRAM.
    
        
    
        
    
          
    
            
    The problem
    
            
    A 128-thread machine is not one flat pool. If a thread on die A touches memory that “belongs” to die B, you pay cross-node latency. Throughput looks capped even though nominal GHz is high.
    
          
    
          
    
            
    What NexusFlow does
    
            
      
              
    • Map hardware (sysfs / hwloc): NUMA nodes, packages, siblings.
      • 
              
    • Pin processes to CPU sets; optional same-node memory bind (numactl).
      • 
              
    • Pipeline with YAML DAG + Prometheus timings; Plasma + shared memory for fast handoffs.
      • 
              
    • Daemon (Linux) cgroup v2 cells, LLC sampling, hugepages, eviction — gRPC.
      •
